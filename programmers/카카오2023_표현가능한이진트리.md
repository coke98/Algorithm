### 💡문제 분석

10진수들을 이진수로 변환했을 때, 이 이진수가 표현 가능한 이진트리인지 확인

**표현 가능한 이진트리 규칙**

- 전처리 (패딩): 10진수를 2진수로 변환한 후, 이진수 문자열의 길이가 2k−1 (포화 이진 트리의 높이에 따른 노드 개수)가 될 때까지 앞에 '0'을 채워 넣어 더미 노드를 추가
- 유효성 검사: 패딩된 이진수 문자열을 트리의 레벨 순서로 간주했을 때, 0인 노드(더미 노드)가 '1'인 자식 노드를 가질 수 없음. 즉, 부모 노드가 '0'이면, 그 서브트리의 모든 노드는 '0'이어야 함

---

### 💡입출력 예시

| `numbers` | `result` |
| --- | --- |
| [7, 42, 5] | [1, 1, 0] |
| [63, 111, 95] | [1, 1, 0] |

---

### 💡코드

```python
def check(bnum):
    if len(bnum) == 1:
        return True 

    idx = len(bnum) // 2
    mid = bnum[idx]
    left = bnum[:idx]
    right = bnum[idx+1:]

    if mid == '0':
        if '1' in left or '1' in right:
            return False
        else:
            return True
    else:
        return check(left) and check(right)

def solution(numbers):
    answer = []
    for number in numbers:
        bnum = bin(number)[2:]
        L = len(bnum)
        k = 1
        
        while (2**k) - 1 < L:
            k += 1
            
        target_L = (2**k) - 1
        padded_bnum = '0' * (target_L - L) + bnum
        result = check(padded_bnum)
        answer.append(1 if result else 0)
        
    return answer
```

---

### 💡시간복잡도

- O(M⋅LlogL)

### 💡공간복잡도

- O(M+L)

---

### 💡메모

- 틀렸던 점
    - 단순히 홀수 길이로 맞추는 것이 아니라, **길이가 2k−1 형태**가 되도록 맞춰야 했음.
    - 루트가 '0'이라고 무조건 실패가 아니라, **'0'인 루트가 '1'인 자손을 가질 때만** 실패여야함.
